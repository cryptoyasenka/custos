## Summary

Posture is solid for a hackathon submission: dedicated SECURITY.md, CodeQL on, npm audit gates prod CVEs in CI, container runs as non-root, secret-scrubbing redactRpcUrl on stdout startup, and the dashboard ships full security headers (CSP, HSTS, X-Frame-Options DENY). Two real concerns: (1) the rotated Helius API key still lives in public git history and at least three planning files in the working tree, so the rotation is mitigation-by-revoke, not erasure; (2) `main` has zero branch protection on a public hackathon repo. Everything else is hardening / defense-in-depth.

### CRITICAL (active leak / live exploit / immediate fix)

- **[git history: commits `deb510d`, `005cd9f`, `6fb6f22`] Helius API key persists in public git history.** Per `.planning/HELIUS-REROLL-RUNBOOK.md` line 13 and `0cc1b82` redaction commit, the leaked key is `2e45da...REDACTED` (length 36, UUIDv4 format). Working-tree file `planning/F3-RECORDING-OPTIONS.md:60` is now placeholder, but the historical commits cannot be purged without rewriting history (which forks already cloned). **Status:** key was rotated 2026-05-09 per Yana's notes — verify `dashboard.helius.dev` shows the old key as `Revoked`. If revoke wasn't actually completed, this is still live.
- **[`.planning/CURRENT.md`, `.planning/HELIUS-REROLL-RUNBOOK.md`] Old leaked key string still present in working tree (tracked).** Both files reference the literal string `2e45da34-dfeb-4bc7-a85c-472e8c16e357`. Even though revoked, the value is now permanently catalogued in two more public files. The `0cc1b82` redaction got `AUDIT-NIGHT-2026-05-09.md` but missed the runbook + CURRENT.md mention.

### HIGH (likely exploitable, not exposing now but fragile)

- **[GitHub repo settings] `main` branch has zero protection.** `gh api .../branches/main/protection` → `404 Branch not protected`. Anyone with push access can force-push, delete the branch, or land unreviewed code. CodeQL is configured but advisories don't block merges. For a public hackathon repo with judges reading the code, recommend at minimum: require PR review (1 approver), require status checks to pass (`ci`, `analyze`), block force-push.
- **[`.github/workflows/ci.yml:13,15`, `codeql.yml:27,30,36`] Third-party Actions pinned to mutable tags, not commit SHAs.** `actions/checkout@v4`, `actions/setup-node@v4`, `github/codeql-action/init@v3`, `.../analyze@v3`. GitHub-owned, so trust is high, but tag pinning means a compromised maintainer (or repo takeover) could repoint `v4` to a malicious commit and the next CI run would execute it. Pin to full SHAs.
- **[npm audit, full incl dev] 4 high severity advisories in dev/test toolchain.** Headline: `bigint-buffer <=1.1.5` (CVE GHSA-3gc7-fjrx-p6mg, CVSS 7.5, buffer overflow in `toBigIntLE()`) reaches `@solana/buffer-layout-utils` → `@solana/spl-token` → `@sqds/multisig`. Suggested fix downgrades `@sqds/multisig` to 1.3.1 (semver-major). Prod-only audit (`--omit=dev`) is clean — these only appear because dev fixtures use `@sqds/multisig`. CI gate is `--omit=dev --audit-level=high` so this won't break CI, but a contributor running `npm install` is exposed at build time.

### MEDIUM (defense in depth, hardening)

- **[`src/alerts/webhook.ts:222-231`] Telegram payload uses `parse_mode: HTML` and interpolates `alert.subject`, `alert.detector`, `alert.cluster`, and `safeStringify(alert.context)` directly into HTML without escaping `<`, `>`, `&`.** Currently safe because every existing detector populates `subject` from `accountBase58` / pubkey strings (base58 alphabet excludes those chars). But fragile: a future detector that puts attacker-controlled strings (token name, DAO display name pulled from token metadata) into the subject would be Telegram-formatted-output corruption / fake-link injection. Add an HTML-escape helper before interpolation, or switch to `parse_mode: MarkdownV2` with proper escaping.
- **[`src/alerts/webhook.ts:94-115`] Discord payload doesn't set `allowed_mentions: { parse: [] }`.** No `content` field is sent (only `embeds`), so today this isn't exploitable — Discord embeds don't ping `@everyone`/`@here` by default. But if anyone later adds a `content:` line that interpolates `alert.subject`, the absence of `allowed_mentions` becomes a real ping-everyone vector. Add explicit `allowed_mentions: { parse: [] }` as a guardrail.
- **[`src/alerts/http.ts:64`] HTTP server binds to all interfaces (`server.listen(this.port, ...)` with no host arg → Node default `::`/`0.0.0.0`).** Required on Railway (edge ingress is internet→container), but a self-host operator who copy-pastes the daemon onto a workstation will expose the alerts API to their LAN unintentionally. README install section should mention this. CORS=`*` and the endpoint is public-by-design (per the comment at line 6-9), so the risk is awareness, not auth bypass.
- **[`Dockerfile:23` (root) + `dashboard/Dockerfile:30`]** Runs as non-root `node` user. Good. Base image is `node:20-alpine` pinned to major tag, not digest — supply-chain best-practice would be `FROM node:20-alpine@sha256:...`.

### LOW / Notes

- **[`.smoke-nonce.json` in working tree]** Contains a 64-byte Solana keypair (devnet ephemeral per `.gitignore:17` comment). Properly gitignored, never committed. Risk is local-machine-only and devnet-scoped. No action needed unless the file accidentally graduates to a non-devnet context.
- **[`custos-nox-session.jsonl` 5.7MB]** Claude Code session log in working tree, gitignored. Never committed. Contains conversation transcripts that may have transient secrets/tokens — recommend grep before sharing the working dir externally.
- **[`.env.example:11,16-19`]** Example values are clearly placeholders (`YOUR_KEY`, `YOUR/WEBHOOK`, `123456789:ABCDEFGhijklmnop`, `-1001234567890`). Clean.
- **[npm audit dashboard]** 2 moderate advisories — `postcss <8.5.10` reaches via `next 16.2.4`. Both moderate, no high/critical. Fine for now; will resolve on next Next.js patch.
- **[`package.json` install-script packages]** Scripts run during `npm ci`: `@biomejs/biome`, `bigint-buffer`, `esbuild` (×2 versions), `fsevents`, `sharp`, `unrs-resolver`. All well-known, none typo-squat suspects.
- **[no `.claude/`, `.cursor/`, `.aider/` dirs in repo]** Clean — no agent prompt-injection surface from external repo contents.
- **[`src/daemon.ts:40-47`] `redactRpcUrl` strips path+query before stdout log.** Good defensive measure given Helius/QuickNode put keys in URL — was added explicitly per commit `e9dc5b5` after recognizing the leak vector.
- **[no `eval`/`Function()`/`child_process` in `src/`, `dashboard/`, `scripts/`]** Clean.
